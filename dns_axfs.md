# CTF: DNS Asynchronous Zone Transfer

__MASTER HANDBOOK 2026.02__

DESCRIPTION: Leverage configuration weaknesses in the subnet's Domain Name Server to dump
domain records. This will reveal other hosts on the network to gain situational
awareness before launching further attacks.

DIFFICULTY: Novice

ESTIMATED TIME: 10 minutes

TARGET: ns1.corp.net (172.31.27.201)

OBJECTIVES:
+ Identify the active network and DNS resolver
+ Perform an unauthorized forward DNS zone transfer (AXFR)
+ Perform a reverse DNS zone transfer (in-addr.arpa)
+ Enumerate DNS service version via CHAOS class queries

## PART 0x00 -- Identify Local Subnet

Before enumerating DNS records, identify which subnet your Attacker machine is connected to.

_Command:_

```
$ ip -4 address show
```

_Expected Output:_

```

    inet 127.0.0.1/8 scope host lo
       valid_lft forever preferred_lft forever
2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc mq state UP group default qlen 1000
    inet 172.31.27.x/24 brd 172.31.27.255 scope global dynamic noprefixroute eth0
       valid_lft 691200 preferred_lft 691200
```

_Network:_

```
172.31.27.0/24
```

## PART 0x01 -- Identify the DNS Search Domain

Automatic DNS resolver configuration can reveal the internal domain name and the IP address of the authoritative DNS server.

_Command:_

```
$ cat /etc/resolv.conf
```

_Expected Output:_

```
# Generated by NetworkManager
search corp.net
nameserver 172.31.27.201
```

## PART 0x02 -- Forward Zone Transfer (AXFR)====

Attempt an unauthorized zone transfer against the **forward lookup domain** using the ''dig'' command. A successful transfer will reveal all known hostnames and records on this network.

_Command(s):_

```
$ dig @ns1.corp.net. AXFR corp.net. # OR
$ dig @172.31.27.201 AXFR corp.net.
```

(in-addr.arpa). Determine the correct **reverse zone** for the 172.31.27.0/24

## PART 0x04 -- DNS Version Enumeration

_Command(s):_

```

$ dig @172.31.27.201 CHAOS TXT version.bind.
```
